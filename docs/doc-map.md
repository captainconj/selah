# Document Map

*What kind of document you are reading, and where to begin.*

Type: `reference`
State: `clean`

---

Selah contains several kinds of documents.

They should not all be read the same way.

## Document Types

### `evidence`

Primary result documents tied closely to computation or rebuilt artifacts.

Use these when you want to know:

- what was measured
- what reproduced
- what currently stands best

Examples:

- [docs/experiments/096-basin-landscape.md](/docs/experiments/096-basin-landscape.md)
- [docs/ark/20-experiment-100.md](/docs/ark/20-experiment-100.md)
- [docs/ark/22-the-map.md](/docs/ark/22-the-map.md)

### `guided reproduction`

Path documents that tell a new reader how to rerun a result or walk a subsystem.

Use these when you want to know:

- how to reproduce a result
- what script to run
- what artifact to inspect

Examples:

- [docs/codex/how-to-walk-the-ark.md](/docs/codex/how-to-walk-the-ark.md)
- [docs/codex/how-to-play-dna.md](/docs/codex/how-to-play-dna.md)
- [docs/start-here.md](/docs/start-here.md)

### `audit / cleanup boundary`

These record what was verified, what was stale, what was fixed, and what still needs care.

Use these when you want to know:

- whether a result was rerun
- whether an artifact was rebuilt
- whether a surface is clean or still mixed

Examples:

- [docs/codex/ark-first-sluice.md](/docs/codex/ark-first-sluice.md)
- [docs/codex/dna-first-sluice.md](/docs/codex/dna-first-sluice.md)
- [docs/codex/tabernacle-first-sluice.md](/docs/codex/tabernacle-first-sluice.md)
- [docs/codex/cleanup-boundary-096-097.md](/docs/codex/cleanup-boundary-096-097.md)

### `synthesis`

Interpretive documents that speak from multiple evidence layers at once.

Use these when you want to know:

- how the project understands the structure as a whole
- how separate findings were brought together

Read these after the evidence and reproduction layers, not before.

Examples:

- [docs/ark/31-the-library-so-far.md](/docs/ark/31-the-library-so-far.md)
- [docs/ark/33-the-genome.md](/docs/ark/33-the-genome.md)
- [docs/tabernacle/13-love.md](/docs/tabernacle/13-love.md)

### `testimony / history`

Documents about how the work unfolded, what it meant to the discoverers, and how the project changed over time.

Use these when you want to know:

- the discovery arc
- the human story
- the spiritual burden and wonder of the process

Read these as history or witness, not as the first evidence layer.

Examples:

- [docs/the-journey.md](/docs/the-journey.md)

### `planning`

Forward-looking documents about what to do next.

Use these when you want to know:

- what remains open
- where cleanup or exploration should go next

Examples:

- [docs/codex/tomorrow-plan.md](/docs/codex/tomorrow-plan.md)
- [docs/codex/make-it-plain-plan.md](/docs/codex/make-it-plain-plan.md)

### `retired / historical`

Older material preserved for honesty or project memory but no longer suitable as a live evidence surface without caution.

Use these when you want project history, not current foundation.

Examples include:

- dead palindrome-era material
- older contaminated artifact discussions
- archive/conversation layers

## Practical Rule

Before trusting any document, ask:

1. what type of document is this?
2. what artifact anchors it?
3. is it reporting, guiding, synthesizing, or witnessing?

That one habit will keep a lot of confusion out of your reading.
